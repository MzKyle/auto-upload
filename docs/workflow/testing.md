# 测试验收流程

## 基础环境验收

- [ ] 应用可以启动
- [ ] 设置页可正常加载默认配置
- [ ] OSS 参数填写后测试连接成功
- [ ] 添加扫描目录后设置会自动保存
- [ ] 任务面板能显示扫描状态和磁盘用量

## 本地上传验收

1. 创建测试父目录：

```bash
mkdir -p /tmp/upload_test/case1
```

2. 放入测试文件：

```bash
echo "hello" > /tmp/upload_test/case1/a.txt
echo '{"ok":true}' > /tmp/upload_test/case1/meta.json
```

3. 在设置中添加 `/tmp/upload_test` 为扫描目录。
4. 触发扫描。
5. 等待稳定性检查完成。

预期：

- [ ] `case1` 中出现 `tmp_upload.json`
- [ ] 任务面板出现上传任务
- [ ] 上传进度从 0 增长到完成
- [ ] `case1` 中出现 `process_task.json`
- [ ] OSS 中出现 `case1/a.txt` 和 `case1/meta.json`
- [ ] 历史记录出现成功记录

## 时间窗口验收

- [ ] 开始和结束都关闭时，任务可立即启动
- [ ] 只启用开始时间时，开始时间前不启动新任务
- [ ] 只启用结束时间时，结束时间后不启动新任务
- [ ] 跨午夜窗口，如 `20:30-06:00`，夜间可启动
- [ ] 已经 uploading 的任务不会被结束时间中断

## 任务控制验收

- [ ] 上传中任务可以暂停
- [ ] 暂停任务可以恢复
- [ ] 失败任务可以重试
- [ ] 取消任务会进入失败状态
- [ ] 重试后已完成文件不重复上传

## 远程机器验收

- [ ] SSH 测试成功
- [ ] rsync 拉取后本地目录出现远程文件
- [ ] rsync 完成后自动创建上传任务
- [ ] SFTP 直传能把远程文件上传到 OSS
- [ ] 停止传输不会导致应用崩溃

## 标注验收

- [ ] 标注窗口可打开
- [ ] 可以选择图片并显示
- [ ] 可以导出 PNG 和 JSON
- [ ] OSS 配置正确时可以上传标注结果
- [ ] 原图属于任务目录时，OSS key 与任务路径一致
