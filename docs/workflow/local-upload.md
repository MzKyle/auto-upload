# 本地目录上传

## 适用场景

本地采集程序将数据写入某个父目录下的批次子目录，应用负责发现这些批次目录并上传 OSS。

## 操作步骤

1. 在“设置”页配置 OSS，并测试连接成功。
2. 在“扫描配置”中添加父级扫描目录。
3. 设置扫描间隔和稳定性检查参数。
4. 根据需要配置上传时间窗口。
5. 回到“任务面板”，点击“触发扫描”或等待自动扫描。
6. 采集程序生成新的子目录。
7. 扫描器等待目录稳定后创建任务。
8. 队列在允许时间内启动上传。
9. 上传完成后在历史记录中查看结果。

## 目录示例

配置扫描目录：

```text
/tmp/upload_test
```

新增批次：

```text
/tmp/upload_test/case1/
  camera1/0001.jpg
  camera1/0002.jpg
  metadata.json
```

OSS prefix 为 `upload/` 时，对象路径：

```text
upload/case1/camera1/0001.jpg
upload/case1/camera1/0002.jpg
upload/case1/metadata.json
```

## 增量说明

当前任务以目录为单位。一旦某个目录已经完成或失败，扫描器不会因为同一路径出现新增文件而自动创建新的任务。如果需要验证“新增文件持续上传”，更可靠的做法是：

- 在目录仍未稳定前持续写入文件，让它们被同一个任务扫描到
- 或为每次新增数据创建新的批次子目录
- 或手动添加该目录并重试对应任务

如果业务要求“已完成目录后续新增文件也自动补传”，需要在扫描器中增加已完成目录的增量检测策略。

## 任务控制

| 操作 | 行为 |
| --- | --- |
| 暂停 | 取消当前任务，运行中文件回退 pending，任务变为 paused |
| 恢复 | 任务变回 pending，等待队列再次启动 |
| 取消 | 任务标记 failed，错误为“用户取消” |
| 重试 | failed 任务变回 pending，未完成文件重新上传 |

## 验证点

- 目录写入未完成时不会立刻生成任务
- 稳定检查通过后出现 `tmp_upload.json`
- 任务上传中出现 `process_task.json`
- OSS 中对象路径符合 prefix/folder/relativePath
- 暂停后可恢复，已完成文件不会重复计数
