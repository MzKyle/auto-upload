# 故障排查 FAQ

## 新目录为什么没有立刻上传？

常见原因：

- 扫描器还没到下一轮扫描
- 目录正在稳定性检查中
- 文件仍在写入，size 或 mtime 持续变化
- 上传时间窗口尚未打开
- 配置的扫描目录不是批次目录的父目录

可以查看任务面板的扫描计划和待稳定目录列表。

## 设置了结束时间，为什么任务还在上传？

结束时间只限制新任务启动。已经处于 `uploading` 的任务会继续执行，直到完成、失败、暂停或取消。

## OSS 测试连接失败怎么办？

按顺序检查：

1. Region 是否填写。
2. Bucket 是否填写正确。
3. AK/SK 是否为空或过期。
4. Endpoint 是否和 Region 匹配。
5. 当前网络是否能访问 OSS。
6. RAM 权限是否允许 list 目标 Bucket。

## 测试连接成功但上传失败？

连接测试只验证 Bucket 可访问，上传还需要写权限。继续检查：

- 是否允许 `PutObject`
- 是否允许分片上传相关权限
- Prefix 权限是否受限
- 上传过程中本地文件是否被删除
- 网络是否出现超时或连接重置

## 任务失败后如何重试？

在任务卡片点击“重试”。系统会把任务状态改为 `pending`，随后队列重新调度。已完成文件会保留完成状态，未完成、失败和中断中文件会重新上传。

## 哪些文件不会上传？

- 隐藏目录中的文件
- `tmp_upload.json`
- `process_task.json`
- 命中黑名单的文件
- 命中正则排除的文件
- 配置了后缀列表但后缀不匹配的文件

白名单命中的文件优先包含。

## rsync 提示命令不存在？

在本机安装 rsync：

```bash
sudo apt-get update
sudo apt-get install -y rsync
```

如果使用密码认证，还需要：

```bash
sudo apt-get install -y sshpass
```

远程机器也需要安装 rsync。

## SFTP 直传很慢或内存高？

当前 SFTP 直传会把单个文件读成 Buffer 后上传。大文件或海量文件建议切换到 rsync 模式，让文件先落盘，再走普通任务上传和 OSS 分片上传。

## 自动清理会删除哪些目录？

只删除满足以下条件的目录：

- 任务状态是 `completed`
- 来源是 `local` 或 `rsync`
- 完成时间早于保留天数截止线
- 目录仍然存在

手动添加的任务不会被自动清理。

## 历史记录删除会删除本地文件吗？

不会。历史删除只删除 SQLite 中的任务记录和关联文件记录。本地文件删除由自动清理服务负责。

## 标注结果上传到哪里？

如果原图属于某个任务目录，会上传到该任务对应的 OSS 路径旁边，并加 `_annotation` 后缀。如果原图不属于任何任务，则使用 OSS 配置里的 prefix 和图片文件名。
