# 数据采集上传工具（`.deb` 包）测试操作文档

> 适用对象：测试工程师  
> 适用环境：Linux（Ubuntu/Debian 系）  
> 目标：从安装到配置 OSS，再到验证“持续跟踪并上传新增文件”的完整流程

---

## 1. 测试准备

### 1.1 测试输入

- 应用安装包：`electron-uploader_2.0.0_amd64.deb`
- 可用的阿里云 OSS 账号信息：
  - `Endpoint`
  - `Region`
  - `Bucket`
  - `AccessKey ID`
  - `AccessKey Secret`
  - （可选）`Prefix`

### 1.2 建议准备的本地测试目录

建议新建如下目录用于测试：

- `/tmp/upload_test/case1`
- `/tmp/upload_test/case2`

并在 `case1` 中先放入几个小文件（如 `.jpg`、`.txt`、`.csv`），后续再追加新文件用于“增量上传”验证。

---

## 2. 安装应用（`.deb`）

在终端进入 `.deb` 所在目录后执行：

```bash
sudo dpkg -i electron-uploader_2.0.0_amd64.deb
```

如果出现依赖缺失，执行：

```bash
sudo apt-get update
sudo apt-get -f install -y
```

然后再次安装（如需要）：

```bash
sudo dpkg -i your-app-name.deb
```

---

## 3. 启动应用

安装完成后，可通过以下任一方式启动：

1. 在系统应用菜单中找到并打开该应用
2. 终端启动（若有命令入口）

进入应用后，默认先打开**任务面板**，请切换到**设置**页进行配置。

---

## 4. 配置阿里云 OSS 并测试连接

进入 `设置` 页面，找到 **阿里云 OSS** 区域，填写以下字段：

- `Endpoint`（示例：`oss-cn-hangzhou.aliyuncs.com`）
- `Region`（示例：`oss-cn-hangzhou`）
- `Bucket`
- `前缀 (Prefix)`（可选，例如 `upload/`）
- `AccessKey ID`
- `AccessKey Secret`

填写后点击 **测试连接** 按钮。

### 4.1 预期结果

- 页面显示：`连接成功`

### 4.2 失败判定

- 页面显示：`失败: ...错误信息...`

### 4.3 失败排查建议

1. `Endpoint` 与 `Region` 是否匹配
2. `Bucket` 名称是否正确
3. AK/SK 是否有对应 Bucket 读写权限
4. 测试机网络是否可访问阿里云 OSS（含 DNS）

---

## 5. 配置扫描目录（文件夹跟踪）

仍在 `设置` 页面，找到 **扫描配置**：

1. 点击 **添加目录**
2. 选择待跟踪的目录（如 `/tmp/upload_test`）
3. 设置 **扫描间隔（秒）**（建议测试时设置较小值，如 `5~10` 秒）

> 说明：应用会定期扫描该目录下子目录，并将符合条件的数据注册为上传任务。

---

## 6. 设置“是否限制时间上传”

在 `设置 -> 上传配置` 中有两个时间项：

- **开始上传时间**
  - 勾选“启用开始时间”后，设置例如 `20:30`
  - 不勾选表示不限制开始时间
- **结束上传时间**
  - 勾选“启用结束时间”后，设置例如 `23:59`
  - 不勾选表示不限制结束时间

### 6.1 关键逻辑（测试重点）

- 时间限制影响“**新任务是否允许启动**”
- **不会中断**已经在进行中的上传任务

---

## 7. 开始上传

可通过以下两种方式触发：

1. 在 `任务面板` 点击 **触发扫描**（立即扫描一次）
2. 等待系统按扫描间隔自动扫描

扫描到可上传目录后，会在任务面板生成任务并进入上传流程（如 `pending/scanning/uploading/completed/failed`）。

---

## 8. 验证“新增文件持续上传”（核心用例）

这是本次测试的关键验收项。

### 8.1 操作步骤

1. 先完成一次基础上传（目录内已有文件上传成功）
2. 向同一跟踪目录新增文件（例如新增 `new_001.jpg`、`new_002.csv`）
3. 等待下一轮扫描，或点击 **触发扫描**
4. 观察任务面板是否出现新上传动作，并检查 OSS 上是否新增对应文件

### 8.2 预期结果

- 新增文件会被继续发现并上传
- 任务状态、文件数量、进度条会更新
- OSS Bucket 中能看到新增对象（路径受 `Prefix` 影响）

## 10. 常见问题（FAQ）

### Q1：为什么没有马上开始上传？

A：可能仍在扫描或稳定性检查阶段；也可能受开始时间限制。

### Q2：设置了结束时间，为什么还在上传？

A：结束时间限制的是“新任务启动”，不会中断正在上传中的任务。

### Q3：测试连接成功但上传失败？

A：优先检查 Bucket 权限策略、对象前缀权限、网络稳定性和文件过滤规则。

### Q4：新增文件没有继续上传？

A：检查是否已配置扫描目录、扫描间隔是否过长、是否触发扫描、时间窗口是否限制了新任务启动。

## 12. 结束标准（通过条件）

满足以下条件可判定主流程通过：

1. 应用可通过 `.deb` 正常安装和启动
2. OSS 可连通且可写入对象
3. 配置目录可被持续扫描并触发上传
4. 可按需启用/关闭时间限制，并符合预期行为
5. 目录后续新增文件可继续自动上传
